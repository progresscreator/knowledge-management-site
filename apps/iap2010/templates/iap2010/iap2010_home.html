MIT IAP 2010 design competition

Knowledge Market Coding Challenge

General Information

First knowledge market coding challenge. The goal is to produce the most useful new features for a knowledge marketplace. Example features include a question and answer exchange, an auction market, a prediction market, a virtual currency used for exchange, and indicators for the health of the knowledge economy. Links to design documents are provided below.

Prizes

  • 1st Place - $5000
  • 2nd Place - $3000
  • 3rd Place - $1000

Important Dates

  1. Registration: midnight January 11, 2010
  2. Initial coding submission: midnight January 18, 2010
    a. This will consist of a URL where judges and other teams can access your code. You may continue to work on your code after this time.
    b. This begins the bug finding phase where you can publicly identify bugs in other teams' code.
  3. Public presentation of features from your code: 13:00 January 22, 2010
    a. This phase is not required but is strongly recommended.
    b. Non-attendance is anticipated only for teams that are non-local.
  4. Final coding submission: midnight January 25, 2010 a. This includes a URL where judges can access your code.
    b. All code should be frozen at this time.
    c. Submission should include a digital writeup/guidebook describing what you have implemented (powerpoint, word or any format you choose).
  5. Final decisions: February 8, 2010

Coding Guidelines

Barter is a prototypical system we are currently building to demonstrate the features and potential applications of a knowledge market. It is still at its early development stage, with a question and answer exchange implemented. The source code of Barter is openly available for you to get started with the code competition. To acquire the latest source code, you need

git clone git://dawei.xvm.mit.edu/socialmobility/knowledge.git

Barter is being implemented using Python under the Django framework. To expediate the development process, we also leverage a popular package called Pinax. You will be able to review customized code under the directory

knowledge/pylib/knowledge/knowledge_web/

Templates and media files are organized on a per-app basis, i.e. each app under the apps/ directory manages its own app-specific templates and media files separately. You are strongly recommended to follow the same organization style, and refer exisiting apps to see how files are structured. While you develop your own world-changing features, you should set up your own database. sqlite3 is recommended because of its light weight and convenience.

To be efficient, you should at least get yourself familiarized with Django tutorials, and some of its advanced features. Meanwhile, it's helpful to be aware of what features are included as part of Pinax. You will need to set up a Pinax environment, instructions can be found here. Remember that you need to activate the pinax environment before you can run python manage.py. Adding some shortcuts and aliases can save your life enormously. We are using Django version 1.1.1, so you also need to upgrade Django, which is included by Pinax, by using

pip install -U Django==1.1.1

when you are under the pinax environment.

There might be some bumps before you get familiar with the current source base. Feel free to email dawei@media.mit.edu if you have any questions or need assistance.
